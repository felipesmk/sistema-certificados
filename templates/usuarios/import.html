Instruções de Importação
  • O arquivo deve ser no formato JSON
  • Usuários existentes serão atualizados (exceto o admin)
  • Novos usuários receberão a senha padrão "123456"
  • Perfis inexistentes serão ignorados
  • O processo é reversível através do histórico
Selecione um arquivo JSON com os dados dos usuários
Exemplo de Estrutura do Arquivo
{
  "exported_at": "2024-01-01T12:00:00",
  "exported_by": "admin",
  "total_users": 2,
  "users": [
    {
      "username": "joao.silva",
      "nome": "João Silva",
      "email": "joao.silva@empresa.com",
      "status": "ativo",
      "role_name": "operador",
      "telefone": "(11) 99999-9999",
      "departamento": "TI",
      "cargo": "Analista",
      "ldap_user": false
    },
    {
      "username": "maria.santos",
      "nome": "Maria Santos", 
      "email": "maria.santos@empresa.com",
      "status": "ativo",
      "role_name": "admin",
      "telefone": "(11) 88888-8888",
      "departamento": "Gerência",
      "cargo": "Gerente",
      "ldap_user": true
    }
  ]
}
Campos Suportados
  • username - Nome de usuário (obrigatório)
  • nome - Nome completo (obrigatório)
  • email - E-mail (obrigatório)
  • status - ativo, inativo ou bloqueado
  • role_name - Nome do perfil
  • telefone - Número de telefone
  • departamento - Departamento
  • cargo - Cargo/função
  • ldap_user - true/false
Observações Importantes
  • Backup: Recomendamos fazer backup antes da importação
  • Senhas: Usuários importados terão senhas resetadas para "123456"
  • Admin: O usuário admin não pode ser alterado via importação
  • Histórico: Todas as alterações ficam registradas no histórico
